Group Members

Briefly describe each member’s strengths and weaknesses as relevant to this project. Your claims will be scrutinised by “intellects vast and cool and unsympathetic” who will not be impressed by boasting. Be particularly clear about how much you know about HTML5, the 'canvas' element, SVG, CSS, and JavaScript.

Section 3 (1%) - Version Control

1% State which version control system you will use and why. Creating accounts on GitHub and SourceForge is easy and free, but you could set up a SubVersion, Mercurial, Git, Darcs, &c repository here or elsewhere. Include the URL of your repository in your report. This gets only 1% because it doesn’t really matter that much which version control system you use as long as you use some version control system.

Section 4 (1%) - Programming Language

1% State what programming language you will use. You may write in raw JavaScript, but you may also use a type checked variant that generates JavaScript, such as TypeScript or Dart. There is no wrong selection here, as far as marks go. The point is to make an informed choice.

Section 5 (1.5%) - Editors and IDEs

1·5% State which editor you will use and why. You should look briefly at Eclipse, NetBeans, BlueFish, Brackets, and WebStorm. All of these except WebStorm are open source, and we have a licence for WebStorm. You should search on the web for comparisons of these programs; while we forbid plagiarism, research is good. The difference is honesty about who did what, giving credit where credit is due. You should include a short list of features that matter to you and why they matter.

Section 6 (1.5%) - Debugging Tools

1·5% Say what debugging tools you propose to use and why. Start with the W3Schools page on JavaScript debugging. You should particularly look at Chrome’s DevTools and Firebug, and of course check back with the IDEs to see what they offer in the way of debugging support.

Section 7 (2%) - Work Schedule

2% End with a section breaking the work to be done into pieces, estimating the pieces, and showing a timetable of who will do what when. You are not required to predict the future perfectly; these are estimates. Do not forget to include the time needed to learn to use your selected tools effectively enough.
