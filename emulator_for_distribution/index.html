Using The Smartwatch Emulator

Emulator Overview

Below is an overview of the smartwatch emulator. This overview is designed to compliment the code documentation and as such specific details of functions, return values etc. are omitted from this text and are instead documented within the code.

This smartwatch emulator has been constructed using a combination of the HTML canvas element, JavaScript and jQuery. The emulator itself has a number of functions for drawing shapes, lines, images and text to the HTML canvas. It has a function to retrieve the current date object or day name, clear the screen or simulate a beep.

Emulator - App Interface

Once the emulator has been initialised a special instruction queue is created called 'instructionQueue' through which all communication to any apps is delivered. An app must poll this queue for incoming instructions. An individual instruction is an array of values. The number of different instructions and their values can be customised to reflect the number of buttons active for the emulator and the type of information associated with each action.

Apps must decode instructions from the queue and decide what to do with the given values.

For example: When the "Swipe Left" button is pressed a new array object is enqueued in the instruction queue with two values. The first values is the string "swipe", the second value is the string "left".

Buttons

There are nine buttons available to the emulator in total. Only six buttons are currently working but extra buttons can be created by adding an ID tag and appropriate button text in the HTML document and adding the 'on click' pattern to the emulator.js file described in the code documentation.

"Activate App" is a special button that activates the app by making a single call to 'main' this is the only time communication is made with the app outside of the instruction queue. For any app to work it must contain a function named 'main' that serves as an entry point to that application.

Screen Taps

In addition to buttons users can interact with the emulator through screen taps. When a user clicks on the emulator screen a new array object is enqueued in the instruction queue with three values. The first value is the string "tap" the second and third values are the X and Y coordinates of that click relative to the emulator screen (i.e. the upper left corner of the screen has the coordinate X = '0', Y = '0').
